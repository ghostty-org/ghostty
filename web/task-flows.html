Ghostties — Task Flows

Use cases, row-click behaviors, and open gaps.
Review before building further.

Lane Structure

Six lanes. Tasks move left-to-right. Backlog and Review lanes are defined but have no click routing yet.

Inbox
status: inbox — unstarted, ready to pick up
Backlog
status: backlog — parked, not urgent
Running
status: running — Claude is working
Needs You
status: needs-you — blocked on you
Review
status: review — PR open
Graveyard
status: done — completed

Flow 1 — Inbox task with project context

The primary happy path. Task arrives from Linear with a known project, user clicks the row, Claude spawns.

flowchart LR
    A([Linear issue]) -->|linear-sync| B[Inbox\ntemplate + project-path set]
    B -->|row click| C[Write status: running]
    C --> D[File-watcher promotes\nrow to Running lane]
    C --> E[Spawn terminal\nGHOSTTIES_TASK_FILE set\norchestrator template]
    E --> F[Claude reads task\nstarts working]
    F -->|gt done| G[Write status: done]
    G --> H[Row → Graveyard]
    G -->|next sync| I([Linear: Done])

    style I fill:#1a2433,stroke:#5b9cf6,color:#e5e5e5
    style E fill:#1a3320,stroke:#4caf7d,color:#e5e5e5

Flow 2 — Orphan task (no project context)

Task arrived from Linear but has no project-path. User assigns one before Claude can spawn.

flowchart LR
    A([Linear issue\nno project]) -->|linear-sync| B[Inbox\nno project-path]
    B -->|row click| C[Orphan triage card\nappears in sidebar]
    C -->|user picks project| D[Writes project-path\nto task .md]
    D -->|chains to Flow 1| E[Spawn terminal\nClaude starts]
    C -->|user dismisses| F[Card closes\nstatus unchanged]

Flow 3 — Create new task (⌘⇧N)

User creates a task manually inside Ghostties. No Linear source required.

flowchart LR
    A([⌘⇧N]) --> B[NewTaskComposerView\nTitle + project]
    B -->|confirm| C[Creates .md in\n.ghostties/tasks/\nstatus: inbox]
    C --> D[File-watcher\npicks it up]
    D --> E[Appears in\nInbox lane]
    E -->|row click| F[Flow 1 or 2]
    B -->|cancel| G[No file written]

    style A fill:#1a2433,stroke:#5b9cf6,color:#e5e5e5

Flow 4 — Click a running task

Task is already running. Click focuses the live session or respawns it if the terminal died.

flowchart LR
    A[Running row click] --> B{Live session\nexists?}
    B -->|yes| C[Focus existing\nClaude session]
    B -->|no — terminal died| D[Respawn at\nproject-path]
    D --> E[Status stays running]

Flow 5 — Needs You task

Claude has flagged the task as blocked on human input.

flowchart LR
    A[Running] -->|Claude sets\nstatus: needs-you| B[Needs You lane]
    B -->|row click| C[Focus or respawn\nClaude session]
    C -->|human responds,\nClaude continues| D[status: running]
    D --> E[Running lane]
    E -->|done| F[Graveyard]

Current State + Gaps

What's built, what's missing, what's coming.

Built

Flow 1 — Standard inbox task

Click inbox row with project-path
  • Status written to disk
  • File-watcher promotes row
  • Terminal spawns with GHOSTTIES_TASK_FILE
  • Orchestrator prompt reads task on start
  • gt done → Graveyard
  • Linear flow-back on next sync
Built

Flow 2 — Orphan triage

Click inbox row without project-path
  • Triage card appears
  • User assigns project
  • Chains to Flow 1 on confirm
  • No dismiss → move to backlog shortcut
Built

Flow 3 — New task (⌘⇧N)

Keyboard shortcut in sidebar
  • Composer opens
  • File written on confirm
  • File-watcher picks it up
  • No template selection in composer
  • No project picker — user types path
Built

Flow 4 — Running task re-click

Click running row
  • Focuses live session
  • Respawns dead session (D8)
  • No indicator which window has the session
Built

Flow 5 — Needs You

Click needs-you row
  • Focus/respawn same as running
  • Claude can't set needs-you status automatically yet
  • No push notification when task needs you
Built

Flow 6 — Graveyard expansion

Click graveyard row
  • Expands to show details
  • Collapses on re-click
  • No restore / reopen action
