# Mission: Ghostties

## Why it exists

Power users running coding agents in parallel are losing the window-management game. Ten Claude sessions, ten terminal tabs, ten context switches. Ghostties puts a row of agent sessions in a sidebar on the terminal they're already using — so you can glance at what's running, click in, and stay in flow.

## Who it's for

**Primary: Sean** — built as a daily driver first. If it works for him, it graduates.

**Secondary: developers running parallel agents** — people already using Claude Code, Codex, or Aider who need a workspace for their agents, not a better terminal.

Not for: casual terminal users, teams, or anyone not on macOS.

## Promise

Open Ghostties in the morning, see what agents ran overnight, know what's waiting — and click in without rebuilding context. The sidebar is the mental model for "what am I working on today."

## For now (not yet, not never)

The terminal engine is upstream Ghostty — untouched. No IDE features, no file tree, no debugger. No team sync, no accounts, no social layer. macOS only. The sidebar is opt-in.

These aren't permanent limits. They're the current scope.

## How to apply

Read this when a feature request arrives or scope creeps. Ask: does this serve the person running parallel agents who needs to stay in flow? If it adds interface for its own sake, it's probably not the next thing.
