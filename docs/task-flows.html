Ghostties — Task Flows

Use cases, row-click behaviors, and open gaps. Review before building further.

Lane Structure

Six lanes. Tasks move left-to-right. Backlog and Review are defined but not yet routed on click.

Inbox
status: inbox
Unstarted, ready to pick up
Backlog
status: backlog
Parked, not urgent
Running
status: running
Claude is working on it
Needs You
status: needs-you
Blocked on human input
Review
status: review
PR open, awaiting review
Graveyard
status: done
Completed tasks

Flow 1 — Inbox task with project context

The primary happy path. Task arrives from Linear with a known project, user clicks, Claude spawns.

flowchart LR
    A([Linear issue]) -->|linear-sync preset| B[Inbox lane\nstatus: inbox\ntemplate: Claude Code\nproject-path: set]
    B -->|row click| C{Has project\npath?}
    C -->|yes| D[Write status: running\nto disk]
    D --> E[File-watcher\npromotes row\nto Running lane]
    D --> F[Spawn terminal\nGHOSTTIES_TASK_FILE exported\norchestrator template]
    F --> G[Claude reads task\nstarts working]
    G -->|user runs gt done| H[Write status: done\nto disk]
    H --> I[File-watcher\npromotes row\nto Graveyard]
    H -->|next linear-sync| J([Linear: Done])

    style A fill:#1a2433,stroke:#5b9cf6,color:#e5e5e5
    style J fill:#1a2433,stroke:#5b9cf6,color:#e5e5e5
    style F fill:#1a3320,stroke:#4caf7d,color:#e5e5e5
    style G fill:#1a3320,stroke:#4caf7d,color:#e5e5e5

Flow 2 — Inbox task without project context (orphan)

Task has no project-path. User must assign one before Claude can spawn.

flowchart LR
    A([Linear issue\nno project]) -->|linear-sync| B[Inbox lane\nno project-path]
    B -->|row click| C{Has project\npath?}
    C -->|no| D[Orphan triage card\nappears in sidebar]
    D -->|user picks project| E[Writes project-path\nto task .md]
    E -->|chains to Flow 1| F[Write status: running]
    F --> G[Spawn terminal\nClaude starts]

    D -->|user dismisses| H[Card closes\nstatus unchanged]

Flow 3 — Create new task (⌘⇧N)

User creates a task manually inside Ghostties. No Linear source required.

flowchart LR
    A([⌘⇧N]) --> B[NewTaskComposerView\nTitle + project]
    B -->|confirm| C[Creates .md file\nin .ghostties/tasks/\nstatus: inbox]
    C --> D[File-watcher\npicks up new file]
    D --> E[Task appears\nin Inbox lane]
    E -->|row click| F[Flow 1 or Flow 2]

    B -->|cancel| G[Dismissed\nno file written]

    style A fill:#1a2433,stroke:#5b9cf6,color:#e5e5e5
    style F fill:#1a2433,stroke:#5b9cf6,color:#e5e5e5

Flow 4 — Click a running task

Task is already in Running lane. Click focuses or respawns the session.

flowchart LR
    A[Running lane\nrow click] --> B{Live terminal\nsession exists?}
    B -->|yes| C[Focus existing\nClaude session]
    B -->|no — session died| D[Respawn terminal\nat project-path\nD8 dead-session rule]
    D --> E[Status stays running\nno lane change]

    style C fill:#1a3320,stroke:#4caf7d,color:#e5e5e5
    style D fill:#1a3320,stroke:#4caf7d,color:#e5e5e5

Flow 5 — Needs You task

Claude is blocked and has flagged the task for human input.

flowchart LR
    A[Running] -->|Claude sets\nstatus: needs-you| B[Needs You lane]
    B -->|row click| C[Focus or respawn\nClaude session\nsame as Flow 4]
    C -->|human responds,\nClaude continues| D[Claude sets\nstatus: running]
    D --> E[Running lane]
    E -->|done| F[Graveyard]

    style B fill:#33280a,stroke:#e5c76b,color:#e5e5e5
    style C fill:#1a3320,stroke:#4caf7d,color:#e5e5e5

Flow 6 — Graveyard task

Completed task. Click expands to show details — no respawn.

flowchart LR
    A[Graveyard row click] --> B[GraveyardRowExpansion\nshows task details]

Current State + Gaps

What's built, what's missing, what's future.

Built

Flow 1 — Standard inbox task

Click inbox row with project-path
  • Status written to disk
  • File-watcher promotes row
  • Terminal spawns with GHOSTTIES_TASK_FILE
  • Orchestrator prompt reads task
  • gt done → Graveyard
  • Linear flow-back on next sync
Built

Flow 2 — Orphan triage

Click inbox row without project-path
  • Triage card appears
  • User assigns project
  • Chains to Flow 1 on confirm
  • No dismiss → backlog shortcut
Built

Flow 3 — New task (⌘⇧N)

Keyboard shortcut in sidebar
  • Composer opens
  • File written on confirm
  • File-watcher picks it up
  • No template selection in composer
  • No project picker UI (types path)
Built

Flow 4 — Running task re-click

Click running row
  • Focuses live session
  • Respawns dead session (D8)
  • No indication which window has the session
Built

Flow 5 — Needs You

Click needs-you row
  • Focus/respawn same as running
  • Claude can't set needs-you automatically yet
  • No notification when task needs you
Built

Flow 6 — Graveyard expansion

Click graveyard row
  • Expands to show details
  • Collapses on re-click
  • No restore/reopen action
